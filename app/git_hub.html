Control de Versiones: Git + VS Code

Sincroniza tu entorno de desarrollo local con repositorios remotos en GitHub de forma ágil.

Requisitos previos obligatorios

Instala previamente Git, configura tu editor VS Code y crea un perfil activo en GitHub.

Paso 01

Verificar la instalación del sistema

Terminal

Despliega la terminal integrada en VS Code usando el atajo de teclado Ctrl + ` e introduce el siguiente comando de verificación:

$ git --version
Si el sistema retorna una cadena similar a git version 2.x.x, tu instalación se encuentra lista.
Paso 02

Configurar credenciales globales de identidad

Git Config

Este proceso se realiza una única vez de forma global. Asegúrate de registrar la misma dirección de correo electrónico que utilizas en GitHub:

$ git config --global user.name "Tu Nombre"
$ git config --global user.email "tucorreo@ejemplo.com"

Para corroborar que los parámetros se guardaron correctamente en el sistema:

$ git config --list
ce small">Terminal

Establece tu identidad global en el sistema. Estos datos se asociarán a cada uno de tus commits públicos:

$ git config --global user.name "Tu Nombre"
$ git config --global user.email "tu@email.com"
Usa el mismo correo electrónico con el que te registraste en tu cuenta de GitHub.
Paso 03

Vincular y subir archivos al repositorio

Terminal

Inicializa tu repositorio local, vincula la URL remota de GitHub y envía tus primeros archivos usando la rama principal:

$ git init
$ git remote add origin https://github.com
$ git branch -M main
$ git add .
$ git commit -m "Mi primer commit"
$ git push -u origin main
Si el sistema te indica que el remoto ya existe, utiliza el comando alternativo git remote set-url origin.
